

# إعداد النشر التلقائي على GitHub Pages

## ملخص
سنضيف ملفات الإعداد اللازمة لنشر الموقع تلقائياً على GitHub Pages باستخدام GitHub Actions. كل مرة يتم فيها تعديل على الكود (سواء من Lovable أو من GitHub)، سيتم بناء الموقع ونشره تلقائياً.

## ما الذي سيحصل؟

1. **إضافة ملف GitHub Actions** (`.github/workflows/deploy.yml`) -- يقوم بتثبيت المكتبات، بناء المشروع، ونشره تلقائياً عند كل تحديث على الفرع الرئيسي.

2. **تعديل إعداد Vite** (`vite.config.ts`) -- إضافة `base` path ليتوافق مع مسار GitHub Pages (مثلاً `/اسم-المستودع/`).

## ما يجب عليك فعله بعد التطبيق

1. افتح مستودع GitHub الخاص بك
2. اذهب إلى **Settings** ثم **Pages**
3. في قسم **Source** اختر **GitHub Actions**
4. انتظر دقيقة أو دقيقتين وسيظهر رابط موقعك

## ملاحظة مهمة
- الخدمات الخلفية (قاعدة البيانات، المصادقة، التخزين) ستظل تعمل من Lovable Cloud لأن مفاتيح الربط مضمّنة بالكود.
- ستحتاج إلى إخباري باسم المستودع على GitHub حتى أضبط المسار بشكل صحيح.

---

## التفاصيل التقنية

### ملف جديد: `.github/workflows/deploy.yml`
- يستخدم Node.js لتثبيت التبعيات وبناء المشروع
- ينشر محتويات مجلد `dist` على GitHub Pages
- يعمل تلقائياً عند كل push على الفرع الرئيسي (main)

### تعديل: `vite.config.ts`
- إضافة خاصية `base` لتتوافق مع المسار الفرعي لـ GitHub Pages
- هذا ضروري حتى تعمل الروابط والملفات بشكل صحيح
